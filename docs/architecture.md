## 1. Jekyll file structure

A standard Jekyll 4 layout with a few deliberate choices in `_config.yml`:

- **`markdown: kramdown`**, **`theme: minima`** (used as a base and heavily
  overridden by `assets/css/style.css`).
- **Collections** (`blog`, `tablaturas`) both set `output: true`. They are used
  instead of Jekyll's built-in `posts` specifically to get **clean, dateless
  URLs** (e.g. `/blog/animes-preferidos/` rather than `/2021/09/20/...`). Source
  lives in `_blog/` and `_tablaturas/`. (A `ded` D&D collection existed
  previously and has been removed.)
- A block of **`url_*` / `user_*` site variables** holds most external links.
  Templates reference them as `{{ site.url_pds }}`, `{{ site.user_spot }}`, etc.
  — change a link in one place. The current set:

  | Variable | Points at |
  |----------|-----------|
  | `github_username` | GitHub (`luccaugusto`) |
  | `url_pds` | `pds.luccaaugusto.xyz` — Papo de Sauna podcast |
  | `url_portfolio` | the site itself / art portfolio root |
  | `url_dev_portfolio` | `dev.luccaaugusto.xyz` — "Meu Computador" dev portfolio |
  | `url_media_server` | `filmes.luccaaugusto.xyz` — self-hosted Jellyfin |
  | `url_mal` / `user_mal` | MyAnimeList |
  | `url_letterboxd` / `user_letterboxd` | Letterboxd |
  | `url_linkedin` | LinkedIn |
  | `user_instagram` | Instagram handle |
  | `user_spot` | Spotify username |
  | `wish_amazon` | Amazon wishlist |
  | `backend_login_url` | `pics.luccaaugusto.xyz/_/#/login` — PocketBase admin |

  (Orkut and the old `url_spot` link var are gone.)

### `_data/` (gallery + content sources)

Gallery pages render from a YAML list of `{ nome, link, thumb }` entries; two
data files feed the global JS instead.

| File | Source of truth | Notes |
|------|-----------------|-------|
| `wallpapers.yml`, `flores.yml`, `suspensao.yml` | **Generated** by `build.sh` | Do **not** hand-edit — regenerated from `images/<dir>/` on every build. |
| `skate.yml` | **Hand-maintained** | No longer regenerated by `build.sh` (the `images/fotos-skate/` photos/clips are listed by hand). |
| `gongo.yml` | **Hand-maintained** | The "Você e o Gongo" image guide. |
| `piadas.yml` | **Hand-maintained** | Joke pool read by `piada.js` (templated in at build time). |
| `splashes.yml` | **Hand-maintained** | Splash-tagline pool read by `splash.js`. |

### Directories at a glance

- `assets/` — `css/`, `js/`, `icons/` (favicons + social/link icons), `img/`
  (UI element images: backgrounds, the back/window buttons, the billboard frame,
  the magazine hand `mao.png`, the joke-telling `caveira-anfitriao.png`, the
  decorative eye/skull, `noise.png`), and **two fonts**: `Ethnocentric-Regular.otf`
  (headings) and `BetterVCR 25.09.ttf` (the retro splash text).
- `cursors/` — `.cur` cursor files (Adventure Time, etc.) read at build time by
  the Liquid-templated `assets/js/random_cursor.js` (no shell generator).
- `images/` — content media (the per-gallery folders `flores/`, `fotos-skate/`,
  `suspensao/`, `wallpapers/`, `gongo/`, plus `blog/` post images and loose
  files). Site-chrome graphics live in `assets/` instead. Generated `thumbs/`
  subfolders are gitignored.
- `downloads/` — downloadable PDFs (Excel course, "Regras Camelô", calendars),
  surfaced by `downloads.html`.
- `docs/` — design specs and this document.
- `scripts/` — `pb-bulk-import.mjs` (Node helper to bulk-seed the PocketBase feed).

## 2. Layouts & includes

### Layouts (`_layouts/`)

| Layout | Body / shell | Used by |
|--------|--------------|---------|
| `home.html` | Minimal: `<html class="zine-home">` + `<body class="palmtree">` + noise overlay + `{{ content }}`. The page supplies its own structure. | `index`, `gongo`, `misc`, `downloads` |
| `post.html` | **Magazine** shell (`<body class="zine-body palmtree">` → `.magazine-layout`): back button, optional rotated `.magazine-title`, `.magazine-content` (multi-column, `.single-column` modifier when `page.single_column`), an "↩ Início" button, and the giant hand `#magazine-holding-hand` (`mao.png`) gripping the corner. | blog posts, `gram`, `treino`, `standup` |
| `fotos.html` | Magazine shell **variant** (`<body class="zine-body">`, no hand): hides the title when it's literally "Fotos" and uses an "↑ Topo" button. | `skate` |
| `gallery.html` | **Billboard gallery** (`<body class="zine-body palmtree billboard-page">`): renders a `.gallery-grid` (with `--gallery-columns`, default 4) **inside the synthwave billboard screen** via `billboard.html`. | `wallpapers`, `suspensao`, `flores` (1 column) |
| `lista.html` | **Zine-panel list** shell (`palmtree` bg + a single `.zine-panel.panel-lista`): back button, crooked-header title, and a star-bulleted `.lista-itens` list of `site[page.collection]` items as title links. | `blog`, `tablaturas` |
| `erro.html` | Minimal error shell (`<body class="erro-404">` + noise + content). | `404` |

Several visual systems coexist: the **magazine** shell (`post`, `fotos`) renders
text/photos as an open magazine on a paper sheet; the **billboard gallery**
(`gallery`) frames a grid inside a perspective-warped synthwave billboard; the
**zine-panel** systems (`home`, `lista`) float `.zine-panel`s on the `palmtree`
background.

> **Note on the magazine "spread".** `style.css` carries a desktop
> `.magazine-page` system (a fixed-height frame whose columns scroll
> *horizontally*) and `assets/js/magazine.js` translates the wheel into sideways
> scroll. Both are **built but currently dormant** — no page applies the
> `magazine-page` body class and no layout loads `magazine.js`, so posts render
> as the normal vertically-scrolling magazine. See
> `docs/2026-06-25-scrolling-magazine-spread.md` to wire it up.

### Includes (`_includes/`)

| Include | Purpose |
|---------|---------|
| `head.html` | `<head>`: meta, `<title>`, favicons, loads `style.css`, and **defers the three global JS files** (`random_cursor`, `piada`, `splash`). Included by every layout. |
| `noise.html` | A fixed, click-through `.overlay.noise` div — the animated film-grain/noise overlay (`noise.png` + a `grain` keyframe). Included by **every** layout. |
| `back.html` | "Back" button: returns to the homepage from `/blog`, otherwise `history.back()` with a referrer check. Used by `post`, `fotos`, `gallery`, `lista`, and a couple of pages directly. |
| `billboard.html` | The synthwave billboard frame (frame image + perspective-warped `.billboard-screen` holding `include.content` + optional scroll-to-top). Loads `billboard-screen.js`. Accepts `content=` and `hide_scroll_to_top=`. Used by the `gallery` layout (wraps the grid) and the homepage (wraps the "Portais" tiles). |
| `last_posts.html` | First 6 blog post titles as links (homepage Blog panel). |
| `download_item.html` | Renders one `<li>` for a downloadable file with a prettified label (strips extension, `-`/`_` → spaces) and an extension badge. Used by `downloads.html`. |
| `window.html` | A draggable "Bem Vindo" desktop-window component with a YouTube embed. **Currently orphaned** — defined but included nowhere (pairs with the dormant `floating-window.js`). |

## 4. CSS styling choices

Three stylesheets, no preprocessor of our own (minima's SCSS is compiled by the
theme; our files are plain CSS).

### Design tokens (synthwave zine palette)

Used consistently across the redesign and the two scoped stylesheets:

| Token | Value | Role |
|-------|-------|------|
| Beige | `#e8d5b7` | Page background of magazine pages |
| Paper | `#f4f0e6` | The "open magazine" sheet |
| Magenta | `#ff2d95` | Primary accent (titles, links-hover, "today") |
| Cyan | `#00f3ff` | Secondary accent (badges, scroll-to-top) |
| Orange | `#f0a830` / `#ff8100` | Header tape strips / neon glows |
| Black | `#000` | Borders + hard offset shadows |

Headings use the **`Ethnocentric`** font (`@font-face`,
`assets/Ethnocentric-Regular.otf`); the homepage splash tagline uses
**`BetterVCR`** (`assets/BetterVCR 25.09.ttf`).

### `assets/css/style.css` (global, ~1,620 lines)

Loaded on every page via `head.html`. It is layered by era — older utility
classes sit alongside the newer redesign systems:

- **Atomic / utility classes (legacy):** colors (`.cor1`–`.cor7`, …), font sizes
  (`.font10`–`.font25`), widths (`.w25`/`.w50`/`.w100`…), alignment helpers, and
  neon glow boxes (`.neon`, `.neon-y`). Class names are Portuguese.
- **Page backgrounds:** body classes set fixed, cover background images —
  `.palmtree`, `.skate`, `.take-me-away`, `.random`.
- **Gallery primitives:** `.galeria`, `.central` (the bordered neon content box),
  `.container-img`, `.imagem` (`object-fit: cover`), `.linha*` flex rows.
- **Zine canvas (homepage):** `.zine-canvas` is the full-viewport stage.
  `.zine-panel`s are styled poster cards; `.crooked-header` is the skewed "taped"
  header strip (star marker + hard `box-shadow`). The interactive panels now live
  in a **two-column flex layout** — `.zine-top` wraps two `.zine-col`s (links on
  the left; blog + fotos on the right) so panels flow inside bounded columns
  instead of free-floating and colliding at mid widths. `.zine-bottom` (formerly
  `.bottom-scene`) flows in below and anchors the billboard scene. See
  `docs/2026-06-26-top-section-responsive-columns.md`.
- **Magazine layout (posts / fotos):** `.zine-body` (beige) centers a
  `.magazine-layout` "paper" sheet that fakes an **open-magazine center fold** via
  a `linear-gradient`. `.magazine-title` is a rotated magenta block with a hard
  shadow; `.magazine-content` flows text in **CSS multi-column** newspaper columns
  (`column-width: 400px`). A desktop `@media (min-width: 809px)` `.magazine-page`
  block turns the sheet into a fixed frame whose columns scroll **horizontally**
  — built but dormant (see the layouts note above).
- **Billboard system:** `.billboard` (fixed, frame image `billboard.png`) +
  `.billboard-screen` warped into the frame's angled display hole via a
  `matrix3d` homography, holding `.billboard-content`. `.billboard-page` makes the
  page a fixed, non-scrolling frame. The homepage "Portais" tiles
  (`.portais`, `.portal-frame`, `.portal-ad-*`) and the gallery `.gallery-grid`
  both render inside this screen. The static `matrix3d` is the no-JS fallback;
  `billboard-screen.js` re-solves it on resize.
- **Splash:** `#splash-text` is a rotated `BetterVCR` tagline anchored to the
  links panel, with a `splash-breath` scale animation, filled daily by `splash.js`.
- **Floating window:** `.window` / `.window_top` / `.windowContent` chrome for the
  orphaned `window.html` component (paired with the dormant `floating-window.js`).
- **Noise overlay:** `.overlay.noise` + a `grain` keyframe drive the film-grain
  layer injected by `noise.html`.
- **Responsive:** `@media (max-width: 808px)` is the general mobile breakpoint;
  the **home canvas breaks at 860px** (stacks the two columns before they get
  cramped, repositions decorative bleed, makes the billboard full width);
  `809px` is the magazine desktop threshold.

### `assets/css/gram.css` (feed, ~135 lines)

Scoped to the photo feed. Provides: a **search bar** (`.gram-search` with icon +
clear button); a responsive 2-col → 1-col `.gram-feed` grid; `.gram-carousel`
using CSS **scroll-snap** with hidden scrollbars; circular overlay buttons
(`.gram-download`, arrows); a `::after` play-triangle hint on video posters; a
fixed-overlay `.gram-modal` lightbox; and a `.magazine-layout` background override
that paints an open-magazine scan (`revista-level-up-aberta-pb.png`,
`color-burn` blend) behind the feed.

### `assets/css/treino.css` (workout sheet, ~130 lines)

Everything scoped under `.treino`. Reskins the workout page into the zine theme:
the wrapper uses **`column-span: all`** to break out of `.magazine-content`'s
newspaper columns and span full width. Recurring motif is the **zine card** =
`2px solid #000` border + `4px 4px 0 #000` hard offset shadow (cards, stats,
table, callouts). `.pill`s are sticker-style badges; today's cloned card gets a
**magenta** offset shadow to read as "this is today". Exercise rows
(`li[data-ex]`) get a caret and toggle an inline `.ex-img`. Grids collapse to one
column under 900px.

## 5. JavaScript

All vanilla, no bundler, no dependencies. `head.html` loads three scripts
**`defer`** on every page; the rest are page-local, and a few exist but aren't
wired in.

### Global (loaded everywhere, via `head.html`)

| File | What it does |
|------|--------------|
| `random_cursor.js` | On load, randomly swaps the page cursor for one of the `.cur` files (with a no-op slot, so the default cursor sometimes stays). **Liquid-templated**: a `site.static_files` loop builds the cursor array from `cursors/*.cur` at build time — don't hand-edit the array. |
| `piada.js` | Injects a random "piada do dia" into `#p-piada` (the homepage skull's speech balloon). **Liquid-templated** — the joke pool comes from `site.data.piadas` (`piadas.yml`), jsonified at build time. Re-runs on click. |
| `splash.js` | Sets a **daily-rotating** crooked tagline in `#splash-text` (same text for everyone on a given UTC day; a birthday override on May 28). **Liquid-templated** from `site.data.splashes` (`splashes.yml`). |

### Page-local (active)

| File | Page / include | What it does |
|------|----------------|--------------|
| `gram.js` | `gram.html` | The **feed renderer**: fetches PocketBase (`window.GRAM_PB_URL`), renders single posts and **carousels** (arrows + dots), images and `<video>`, light thumbnails upgrading to full-res in a **modal lightbox**, per-item **download** buttons, **search** (tag/description, debounced), and "Carregar mais" pagination. |
| `treino.js` | `treino.html` | Clones the workout card matching the **viewer's local weekday** to a "Hoje —" block at the top; shows a rest-day note on weekends. The week grid stays the source of truth. |
| `treino-images.js` | `treino.html` | Makes each `li[data-ex]` a focusable toggle that lazy-loads and shows/hides an exercise image (`/assets/img/treino/<slug>.jpg`); supports click and keyboard, tracks `aria-expanded`. |
| `billboard-screen.js` | `_includes/billboard.html` | Re-solves the `.billboard-screen` perspective homography on resize so the screen stays pinned to the billboard's angled hole at any size. The static `matrix3d` in `style.css` is the no-JS fallback. Loaded wherever `billboard.html` is (gallery pages + homepage). |

### Built but not wired in

| File | Status |
|------|--------|
| `magazine.js` | Sideways-scroll reader for the dormant `.magazine-page` spread — **loaded nowhere** (see the §2 magazine note). |
| `floating-window.js` | Draggable window chrome for the orphaned `window.html` — **loaded nowhere**. |
| `billboard-corner-pin.js` | A dev tool: activate with `?pin` to drag corner handles and read out the `matrix3d` to bake into `style.css`. Not loaded in normal page flow; intended for deletion once values are baked. |

> **Note:** `gram.html` previously shipped a TEST-DATA `<script>` that mocked the
> PocketBase API. It has been **removed** — the page now sets
> `window.GRAM_PB_URL = "https://pics.luccaaugusto.xyz"` and `gram.js` fetches the
> live backend directly.

## 6. Plugins

Declared in `Gemfile` / `_config.yml`:

- **`jekyll-feed`** — generates the Atom feed at `/feed.xml`.
- **`jekyll-loading-lazy`** — automatically adds `loading="lazy"` to images at
  build time (galleries lean on this).
- **`minima` (~2.5)** — the base theme. Mostly overridden by `style.css`, but it
  still transitively pulls in **`jekyll-seo-tag`** and **`jekyll-sass-converter`**
  (SCSS compilation).
- **`kramdown`** — the Markdown engine for posts/collections.

`webrick`, `logger`, and `bigdecimal` are in the Gemfile to satisfy Ruby 3.4+ /
Jekyll-serve runtime needs, not as content plugins.

## 7. Integrations & external services

- **PocketBase** (`pics.luccaaugusto.xyz`) — a self-hosted single-binary backend
  (SQLite + auth + admin UI + file storage + REST API) powering the dynamic photo
  feed. The static site fetches it client-side at view time, so posts publish
  instantly without a rebuild. Full design & ops in
  `docs/2026-06-04-instagram-feed-design.md` and `-plan.md`;
  `scripts/pb-bulk-import.mjs` seeds it from a folder of media. Admin login is
  `backend_login_url`.
- **GitHub Actions → VPS** — `ci.yml` (on push to `main`) installs ImageMagick,
  sets up Ruby 4.0, runs `build.sh`, then `rsync`s `_site/` over SSH to
  `/var/www/blog.luccaaugusto.xyz/html/`.
- **Embeds** — a standup MP4 (`standup`), Spotify/Deezer (Gongo podcast), and a
  YouTube embed inside the orphaned `window.html`.
- **Profile/links** — GitHub, a self-hosted Jellyfin (`url_media_server`),
  Letterboxd, MyAnimeList, LinkedIn, Instagram, Spotify, and an Amazon wishlist
  (see the `url_*` / `user_*` table in §1).
- **Sibling subdomains** — `pds.luccaaugusto.xyz` (Papo de Sauna),
  `dev.luccaaugusto.xyz` (dev portfolio / "Meu Computador"),
  `filmes.luccaaugusto.xyz` (Jellyfin), `pics.luccaaugusto.xyz` (PocketBase).
