# AGENTS.md

Guide for AI agents (and humans) working in this repository.

`luccaaugusto.xyz` — Lucca Augusto's personal site and blog. A **Jekyll 4** static
site with a **synthwave / zine** aesthetic, Portuguese (pt-BR) content, and a
playful, irreverent tone. Built and deployed by GitHub Actions; one page (the
photo feed) is dynamic and talks to a self-hosted PocketBase backend.

For the full reference (CSS system, JS modules, every page, integrations), read
**[docs/architecture.md](docs/architecture.md)**. This file is the short version.

## 🐳 Golden rule: everything runs in Docker

**Run and verify every command inside the Docker container — never on the host.**
The host has no working Ruby/Bundler, and `_site/` ends up root-owned from
container runs, so host builds fail on permissions.

```bash
docker compose up                 # start the dev server (foreground)
docker compose up -d              # ...or detached
docker compose exec site <cmd>    # run any command in the container
docker compose restart site       # restart (see _config.yml note below)
```

- Dev server: `bundle exec jekyll serve` on **http://localhost:4004** with
  `--livereload` (`--force_polling`, so edits picked up on a mounted volume).
  The service is named `site` (container `site-site-1`). Image: `ruby:4.0`.
- Gems are cached in the `bundle_cache` volume (outside the project tree) so
  repeated `up` runs are fast.
- **`_config.yml` is NOT hot-reloaded.** After editing it, run
  `docker compose restart site`.
- Prefer a single check over long polling/verification loops.

## Build & deploy

- **`build.sh`** is the production build (run by CI; also runnable in-container):
  1. Regenerates the gallery data files + thumbnails (see *Galleries* below).
  2. Stamps `Atualizado em <date>` into any changed `_blog/*.md`.
  3. `JEKYLL_ENV=production bundle exec jekyll build`.
  4. Strips `*.sh` out of `_site/`.
- **CI** (`.github/workflows/ci.yml`): on every `push`, installs ImageMagick,
  sets up Ruby 4.0, runs `build.sh`, then `rsync`s `_site/` to the VPS over SSH.
- ImageMagick (`convert`) is a build dependency — it generates gallery thumbnails.

## Plugins & integrations

- **Plugins:** `jekyll-feed` (Atom `/feed.xml`), `jekyll-loading-lazy`
  (auto `loading="lazy"` on images), `minima` theme (heavily overridden;
  pulls in `jekyll-seo-tag`), `kramdown` Markdown, SCSS via `jekyll-sass-converter`.
- **PocketBase** (`pics.luccaaugusto.xyz`): self-hosted backend for the photo
  feed (`gram.html`). `gram.html` sets `window.GRAM_PB_URL` and `gram.js` fetches
  it live. See `docs/2026-06-04-instagram-feed-*.md`.
- **External links** live as `url_*` / `user_*` vars in `_config.yml` (GitHub,
  Jellyfin `filmes.`, Letterboxd, MyAnimeList, LinkedIn, Instagram, Spotify,
  Amazon wishlist, sibling subdomains `pds.`/`dev.`). YouTube/Spotify/Deezer and
  a standup MP4 embed appear on a few pages.

## Conventions & gotchas

- **Language:** content *and* CSS class names are Portuguese
  (`.galeria`, `.central`, `.linha`, `cabecalho`, `piada`, `treino`).
- **Galleries are data-driven.** Drop files in `images/<gallery>/` and run
  `build.sh`; it regenerates `_data/{wallpapers,flores,suspensao}.yml` and
  thumbnails. **Do not hand-edit those generated files.** `_data/{skate,gongo}.yml`
  are the exceptions — maintained by hand (no longer regenerated).
- **`random_cursor`, `piada`, and `splash` JS are Liquid-templated.**
  `random_cursor.js` builds its cursor list from `cursors/*.cur` via
  `site.static_files`; `piada.js` / `splash.js` pull their pools from
  `_data/piadas.yml` / `_data/splashes.yml`. Edit the source files, not the arrays.
- **Some JS/CSS is built but dormant.** The magazine sideways-scroll spread
  (`.magazine-page` CSS + `magazine.js`) and the `window.html` / `floating-window.js`
  desktop window exist but are wired into no page. Don't assume they're live.
- No automated test framework — verify changes by loading pages in the running
  container (`http://localhost:4004`).
