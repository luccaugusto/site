## ­¤É│ Golden rule: everything runs in Docker

**Run and verify every command inside the Docker container ŌĆö never on the host.**
The host has no working Ruby/Bundler, and `_site/` ends up root-owned from
container runs, so host builds fail on permissions.

```bash
docker compose up                 # start the dev server (foreground)
docker compose up -d              # ...or detached
docker compose exec site <cmd>    # run any command in the container
docker compose restart site       # restart (see _config.yml note below)
```

- Dev server: `bundle exec jekyll serve` on **http://localhost:4000** with
  `--livereload` (`--force_polling`, so edits picked up on a mounted volume).
  The service is named `site` (container `site-site-1`). Image: `ruby:4.0`.
- Gems are cached in the `bundle_cache` volume (outside the project tree) so
  repeated `up` runs are fast.
- **`_config.yml` is NOT hot-reloaded.** After editing it, run
  `docker compose restart site`.
- Prefer a single check over long polling/verification loops.

## Build & deploy

- **`build.sh`** is the production build (run by CI; also runnable in-container):
  1. Regenerates the gallery data files + thumbnails (see *Galleries* below).
  2. Stamps `Atualizado em <date>` into any changed `_blog/*.md`.
  3. `JEKYLL_ENV=production bundle exec jekyll build`.
  4. Strips `*.sh` out of `_site/`.
- **CI** (`.github/workflows/ci.yml`): on every `push`, installs ImageMagick,
  sets up Ruby 4.0, runs `build.sh`, then `rsync`s `_site/` to the VPS over SSH.
- ImageMagick (`convert`) is a build dependency ŌĆö it generates gallery thumbnails.

## Plugins & integrations

- **Plugins:** `jekyll-feed` (Atom `/feed.xml`), `jekyll-loading-lazy`
  (auto `loading="lazy"` on images), `minima` theme (heavily overridden;
  pulls in `jekyll-seo-tag`), `kramdown` Markdown, SCSS via `jekyll-sass-converter`.
- **PocketBase** (`gram.luccaaugusto.xyz`): self-hosted backend for the photo
  feed (`gram.html`). See `docs/2026-06-04-instagram-feed-*.md`.
- **External links** live as `url_*` vars in `_config.yml` (GitHub, Spotify,
  Letterboxd, MyAnimeList, Orkut, LinkedIn, Amazon wishlist, sibling subdomains
  `pds.`/`dev.`). YouTube/Spotify/Deezer embeds appear on a few pages.

## Conventions & gotchas

- **Language:** content *and* CSS class names are Portuguese
  (`.galeria`, `.central`, `.linha`, `rodape`, `cabecalho`, `piada`, `treino`).
- **Galleries are data-driven.** Drop files in `images/<gallery>/` and run
  `build.sh`; it regenerates `_data/{skate,wallpapers,flores,minhas_fotos,suspensao}.yml`
  and thumbnails. **Do not hand-edit those generated files.**
  `_data/gongo.yml` is the exception ŌĆö maintained by hand.
- **`gram.html` ships a TEST-DATA `<script>`** that mocks the PocketBase API with
  placeholder media. **Delete/comment that block before relying on the live feed.**
- **`assets/js/random_cursor.js` is Liquid-templated** ŌĆö it loops over
  `site.static_files` to build its cursor list from `cursors/*.cur` at Jekyll
  build time. Add/remove `.cur` files in `cursors/`; don't hand-edit the array.
- No automated test framework ŌĆö verify changes by loading pages in the running
  container (`http://localhost:4000`).
